Section 4: An In-depth look at the API URL Address

How to get different stocks or stock properties from the Yahoo API

Remember in section 2 when we executed a GET call with the URL: (http://download.finance.yahoo.com/d/quotes.csv?s=GOOG&f=nsl1&e=.csv) ?

This was actually a call to Yahoo's North American servers to get us the specifed data.

Breakdown of the URL:

The first parameter in the GET call is designated as "s". This parameter lets you designate which stocks you want to get information for by inputting the stock symbol. You can designate multiple stocks by using a comma.

Example 1:

s=GOOG

This specifies that you want information for 1 stock. Here, "GOOG" stands for google.

Example 2:

s=GOOG,YHOO

This specifies that you want information for 2 stocks. Here, "GOOG" stands for Google and "YHOO" stands for Yahoo.

The second parameter in the GET call is designated as "f". This parameter lets you designate which proporties to get for the specified stock. A full property list can be found here. The properties are not delimited by any character. You can add properties in any order.

Example 3:

f=nsl1

The property order in this example is stock name(s), stock symbol(s), latest quote of the stock(l1). Notice here that stock name and stock symbol are designated with "n" and "s", not "n0" and "s0". When you designate a property without using a number as the second character, then API will get the property that corresponds to the lowest integer of that letter. For instance, you input "x" into the properties paramater. The API will return the stock exchange of the stock; this is because the API will look for "x0".

Example 4:

f=abb4g3

The property order in this example is ask price(a), bid price(b), book value per share(b4), and annualized gain(g3). This shows the property separation more clearly: f=[a][b][b4][g3]. Notice, again that "a" and "b" is interpreted as "a0" and "b0"

The third parameter in the GET call is designated as "e". This property is constant. By keeping this property set to ".csv", we are getting a csv formatted string every time we call this API.
